## ðŸ”· Technical Architecture (Hedera-Specific)

### **Smart Contracts on Hedera**

**Core Contracts**:
1. **InsurancePool.sol**: Manages staking, yield allocation, claim payments
   - Receives deposits (YHT, YBOB-H, YGOLD-H, GAMI)
   - Deploys to yield farming (BTC.HBAR LP, validator staking)
   - Executes claims within 3-6 seconds
   - Gas cost: ~0.001 HBAR per interaction

2. **ClaimsValidator.sol**: AI integration point
   - Receives claim request (user ID, claim type, amount, evidence)
   - Calls ZK proof verification
   - Calls AI risk scoring engine
   - Approves/denies claim or flags for DAO vote
   - Settles payout to user/insurance provider

3. **GovernancePool.sol**: DAO voting
   - Enables token-weighted voting on pool parameters
   - Timelock enforcement (48 hours for major changes)
   - Multi-sig execution (3-of-5 signers required)
   - Transparent vote logging on Hedera ledger

4. **YieldFarming.sol**: Integrates with Hedera DEXs
   - Deposits pool capital into BTC.HBAR LP (Pangolin, SaucerSwap)
   - Harvests LP fees + HBAR staking rewards
   - Compounds interest automatically
   - Rebalances on demand or scheduled (daily/weekly)

5. **ZKProofVerifier.sol**: Privacy layer
   - Verifies zk-SNARK proofs (medical claims, death certificates)
   - Never stores raw data on-chain
   - Only stores proof hashes
   - Hedera Consensus Service ensures proof immutability

### **Off-Chain Infrastructure**

**1. Agentic AI Engine**
- **Language**: Python (backend) + Node.js (API)
- **Hosted**: Decentralized nodes (community-run, incentivized with KAI tokens)
- **Functions**:
  - Claims risk scoring (ML model trained on historical data)
  - Provider API integrations (NHIF, Equity, APA calls)
  - Micro-agent orchestration (M-Pesa, USSD, WhatsApp)
  - Education nudges (personalized recommendations)
- **Privacy**: All models trained on anonymized data; no PII in model weights

**2. Oracle Integration**
- **Chainlink**: For external data (BTC.HBAR price, weather data, crop indices)
- **Hedera Native Oracles**: For Hedera-specific data (HBAR price, validator APY)
- **Custom Oracles**: Healthcare data (hospital discharge records via encrypted APIs)
- **Update Frequency**: Real-time for prices; daily for health/agricultural data

**3. Zero-Knowledge Proof System**
- **Framework**: Circom (circuit language) + Plonk (proof system)
- **Proofs**:
  - Medical: "Patient hospitalized, matches policy terms" (without diagnosis)
  - Agricultural: "Crop health index below threshold" (without yield data)
  - Death: "Person deceased on date X, age >18" (without cause/location)
- **Verification**: On-chain smart contracts verify proofs in <100ms

**4. Storage**
- **On-Chain (Hedera)**: Claim status, payout records, governance votes, proof hashes
- **Off-Chain (IPFS)**: Encrypted medical records, farm data, business documents
- **Cold Storage**: Backup reserves in hardware wallets (3-of-5 multi-sig)
- **Database**: PostgreSQL for indexing claims, fast queries for DAO dashboards
